## Component Inventory

New UI elements introduced in Phase 5:

### `.article-meta` — Metadata line

**Placement:** Inside `devliot-article-page`, rendered above `<article>` body content, below the article title area.

**Structure:**
```html
<p class="article-meta">April 11, 2026&nbsp;·&nbsp;5 min read</p>
```

**CSS contract:**
```css
.article-meta {
  font-size: var(--font-size-label);   /* 14px */
  font-weight: var(--font-weight-regular);  /* 400 */
  color: var(--color-text-muted);      /* #666666 */
  line-height: 1.5;
  margin-bottom: var(--space-md);      /* 16px — before article body */
  margin-top: 0;
}
```

**Separator:** `&nbsp;·&nbsp;` (middle dot with non-breaking spaces, both sides). No visual separator element.

**Conditional rendering:**
- Date portion: rendered only when `_date` is non-empty
- Reading time portion: rendered only when `_readingTime > 0`
- If only date is available (no reading time yet): renders "April 11, 2026" without the separator
- If neither: the `<p>` element is not rendered at all

**Date format:** `Intl.DateTimeFormat('en-US', { year: 'numeric', month: 'long', day: 'numeric' })` — produces "April 11, 2026". Source: CONTEXT.md D-05.

**Reading time format:** `{N} min read` — integer, no decimals. Minimum display: "1 min read". Source: CONTEXT.md D-06.

### OG HTML pages (`dist/articles/{slug}/og.html`)

Not a visual UI component — crawler-only artifacts. No styling contract required. Human visitors are immediately redirected to `/#/article/{slug}` via inline `<script>`. The `<body>` is empty.

**OG image spec (author-provided, not generated):**
- Filename: `og-image.png` per article directory
- Dimensions: 1200 × 630 pixels (aspect ratio 1.91:1)
- Format: PNG preferred for text/graphics; JPEG for photos
- Max size: < 500KB recommended (Facebook limit is 8MB)
- Path in `index.json`: `"image": "articles/{slug}/og-image.png"` (relative from site root)
- Absolute URL in OG tag: `https://devliot.github.io/devliot/articles/{slug}/og-image.png`
- When missing: omit `og:image` and `twitter:image` tags entirely — do not emit empty `content=""`

## Accessibility Contract

| Concern | Contract |
|---------|---------|
| Metadata line semantics | Use `<p class="article-meta">` — paragraph element, not a heading or ARIA landmark |
| Reading time aria | No `aria-label` needed — "5 min read" is self-describing plain text |
| Date format | `Intl.DateTimeFormat` produces screen-reader-friendly "April 11, 2026" — no `<time>` element required, but recommended: `<time datetime="2026-04-11">April 11, 2026</time>` |
| OG pages | Crawler-only artifacts. Empty `<body>`. No accessibility requirements. |
| Focus management | No new focus targets introduced in this phase |

**Recommended `<time>` element for date:**
```html
<time datetime="${this._date}">${this._formatDate(this._date)}</time>
```
This provides the ISO date as `datetime` attribute for accessibility tools while showing the long-format date visually.

## index.json Schema Contract

Current schema (as found in `public/articles/index.json`):
```json
{
  "articles": [
    {
      "slug": "01-demo-article",
      "title": "Article Components Demo",
      "date": "2026-04-11",
      "category": "Tutorial",
      "tags": ["demo", "components", "reference"]
    }
  ]
}
```

Extended schema for Phase 5 (new fields: `description`, `image`, `readingTime`):
```json
{
  "articles": [
    {
      "slug": "01-demo-article",
      "title": "Article Components Demo",
      "date": "2026-04-11",
      "category": "Tutorial",
      "tags": ["demo", "components", "reference"],
      "description": "Author-written summary ~160 chars for OG/Twitter Card social sharing.",
      "image": "articles/01-demo-article/og-image.png",
      "readingTime": 5
    }
  ]
}
```

- `description`: author-written, max 160 chars, required for META-01
- `image`: relative path from site root, optional (OG `og:image` tag omitted when absent)
- `readingTime`: integer minutes, injected by `build-og-pages.mjs` at build time — authors do NOT write this field

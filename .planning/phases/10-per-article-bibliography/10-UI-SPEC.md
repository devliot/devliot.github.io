## Typography

**Reference entry typography decision (from Claude's Discretion, CONTEXT.md):**
Reference entries use `--font-size-label` (14px) at `--font-weight-regular` (400) with `--line-height-body` (1.5). This is consistent with `.article-meta` and `.article-byline` — metadata-class content, visually subordinate to the article body.

## Color

All tokens already declared in `src/styles/reset.css`. Monochrome palette — no colored accent.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (`--color-surface`) | Page background, article surface |
| Secondary (30%) | #f8f9fa (`--color-surface-alt`) | Inline code backgrounds — no new use in this phase |
| Accent (10%) | #333333 (`--color-accent`) | Heading anchor `#` icon, blockquote border, tag hover, figcaption prefix — NOT citation links |
| Text muted | #666666 (`--color-text-muted`) | Citation links `[N]` color, reference entries, ↩ back-link |
| Border | #e5e5e5 (`--color-border`) | Separator `border-top` above references section (same as `.article-tags` uses `#eeeeee` — use `--color-border` for consistency) |
| Destructive | not applicable | No destructive actions in this phase |

**Accent reserved for:** heading anchor `#` icons, blockquote left border, tag-link hover state, figcaption "Figure N:" prefix. Citation links are explicitly NOT accent — they use `--color-text-muted` to remain visually muted (CONTEXT.md D-10).

**Citation link color rule (CONTEXT.md D-10):** Inline `[N]` links use `--color-text-muted` (#666666) with underline on hover only. This differentiates them from standard content links (which use `--color-accent` + permanent underline per `article.css` line 150-153).

Source: CONTEXT.md D-10 + `src/styles/reset.css` (pre-populated).

## Interaction Contract

All interactions defined by CONTEXT.md decisions. No new patterns needed.

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Click inline `[N]` | User clicks citation link in article body | Smooth-scrolls to corresponding `.ref-entry` in "Références" section; respects `scroll-margin-top` sticky-header offset |
| Click ↩ back-link | User clicks back-link at end of a reference entry | Smooth-scrolls back up to the inline `[N]` citation in the article body |
| Click reference title link | User clicks the title text of a reference with a `url` | Opens URL in new tab (`target="_blank" rel="noopener noreferrer"`) |
| Hover `.citation-link` | Pointer enters inline `[N]` | Underline appears (was none at rest) |
| Hover `.ref-backlink` | Pointer enters ↩ | Underline appears (was none at rest) |

Scroll behavior: `scrollIntoView({ behavior: 'smooth' })` — identical to Phase 7 heading anchor pattern. Source: CONTEXT.md D-12, D-13, D-14.

## Layout Contract

**Placement in article page** (CONTEXT.md D-05):

```
<article>  ← article body (unsafeHTML)
  ...body content with inline [N] citations...
</article>
<section class="references">          ← NEW: references section
  <h2>Références</h2>
  <ol class="ref-list">
    <li class="ref-entry" id="ref-1">
      [1] Authors — Title. Publisher, Year. <a>↩</a>
    </li>
  </ol>
</section>
<nav class="article-tags">           ← existing tags nav (unchanged)
```

**Separator:** `border-top: 1px solid var(--color-border)` + `padding-top: var(--space-xl)` + `margin-top: var(--space-xl)` — mirrors `.article-tags` separator pattern exactly.

**No references case:** When `bibliography` is absent or empty, the `.references` section is not rendered. No placeholder, no separator. Source: CONTEXT.md D-06.

## Reference Entry Format

Compact single-line format per type (CONTEXT.md D-03, D-07, D-08):

| Type | Format |
|------|--------|
| `article` | `[N] Authors — Title. Year.` (title is link if `url` present) |
| `book` | `[N] Authors — Title. Publisher, Year.` (title is link if `url` present) |
| `web` | `[N] Title.` (title is link — almost always has `url`) |

Multiple authors: joined with `", "` — no auto "et al." generation. Author strings rendered as provided in data. Source: CONTEXT.md D-08.

## Anchor ID Naming (Claude's Discretion)

Resolved from CONTEXT.md discretion list and Phase 7 `cite-{n}` / `ref-{n}` pattern convention:

| Element | ID pattern | Example |
|---------|-----------|---------|
| Inline citation `[N]` | `cite-{N}` (1-based index) | `id="cite-1"` |
| Reference entry | `ref-{N}` (1-based index) | `id="ref-1"` |

Use 1-based numeric IDs (not the raw `id` string from `BibliographyEntry`) to keep anchors stable regardless of entry `id` field content. Source: Claude's Discretion (CONTEXT.md).

## Component Inventory

### New: `.article-byline` paragraph

**Element:** `<p class="article-byline">` inserted immediately after `.article-meta` in `devliot-article-page.ts render()`.

**Always rendered** тАФ never conditional. When `authors` array is empty or absent, renders the default `par Devliot` fallback (D-04).

**CSS spec:**
```css
.article-byline {
  font-size: var(--font-size-label);    /* 14px */
  font-weight: var(--font-weight-regular); /* 400 */
  color: var(--color-text-muted);       /* #666666 */
  line-height: 1.5;
  margin-bottom: var(--space-md);       /* 16px тАФ matches .article-meta bottom margin */
  margin-top: 0;
  padding-left: var(--space-lg);        /* 24px тАФ matches .article-meta horizontal padding */
  padding-right: var(--space-lg);
}
```

**Author link spec:** Author names with a `url` field render as:
```html
<a href="{url}" target="_blank" rel="noopener">{name}</a>
```
These inherit the global `a:not(.heading-anchor)` rule тАФ no additional CSS class required. No new link styling needed.

**Author name without url:** rendered as plain text inline (no `<span>`, no styling).

Source: CONTEXT.md D-02, D-03, D-04; `src/styles/article.css` (.article-meta pattern)

## Interaction Contract

### Byline rendering logic

| Authors array state | Rendered output |
|--------------------|-----------------|
| Empty array `[]` or field absent | `par <a href="https://github.com/devliot" target="_blank" rel="noopener">Devliot</a>` |
| Single author with url | `par <a href="{url}" target="_blank" rel="noopener">{name}</a>` |
| Single author without url | `par {name}` |
| Two authors | `par {A} et {B}` (each formatted per url rule) |
| Three or more authors | `par {A}, {B} et {C}` (comma-separated, `et` before last) |

**Separator pattern:** `par ` prefix (with trailing space), authors joined with `, ` except last separator which is ` et `.

Source: CONTEXT.md D-01, D-03, D-04

### No interactive states for the byline container

The `.article-byline` paragraph itself has no hover, focus, or active state. Only the `<a>` elements inside it carry standard link interaction (underline always visible, browser default focus ring).

### JSON-LD structured data (non-visual)

The JSON-LD `<script type="application/ld+json">` block is placed in `public/articles/{slug}/og.html` at build time. It has no visual rendering тАФ it is invisible to users. No visual contract needed for this element.

Source: CONTEXT.md D-06, D-07, D-08

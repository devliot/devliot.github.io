## Spacing Scale

Declared values (CSS custom properties from `reset.css` — unchanged from Phase 2):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, inline code padding |
| `--space-sm` | 8px | Tight element spacing (copy button padding, language badge) |
| `--space-md` | 16px | Paragraph gap, code block internal padding |
| `--space-lg` | 24px | Article horizontal padding, between heading and paragraph |
| `--space-xl` | 32px | Between major content blocks (minimum) |
| `--space-2xl` | 48px | Between major content blocks (maximum, e.g. diagram → next section) |
| `--space-3xl` | 64px | Page-level separation (not used inside article body) |

Source: CONTEXT.md D-14 ("32-48px between major blocks, 16-24px between paragraphs"), `src/styles/reset.css`.

Exceptions:
- Inline code: `2px 4px` padding (2px vertical is a sub-4px exception, carried from Phase 2 `reset.css`)
- Copy button touch target: minimum 44px × 44px (accessibility requirement, D-06)
- Figure caption: `--space-sm` (8px) between image bottom edge and caption text

## Typography

All values from `src/styles/reset.css` — pre-populated, no changes for Phase 3.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | `--font-size-body` | 16px | 400 (`--font-weight-regular`) | 1.5 (`--line-height-body`) | Article prose paragraphs |
| Label | `--font-size-label` | 14px | 400 (`--font-weight-regular`) | 1.4 | Language badge, figure caption text, "Copied!" feedback |
| Heading | `--font-size-heading` | 28px | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-heading`) | Article `<h1>` |
| Sub-heading | — | 20px | 600 (`--font-weight-semibold`) | 1.2 | `<h2>`, `<h3>` within article body |

Notes:
- Sub-heading (20px) is a new declaration for Phase 3. `<h2>` and `<h3>` are distinct from the global `--font-size-heading` token because article sections require visual hierarchy below the page title. `<h4>` and below use 16px body size at semibold weight.
- Monospace font (`--font-family-mono: 'Fira Code'`) used exclusively for code blocks. Font size: 14px (`--font-size-label`) to maintain compact code density at normal zoom.

Source: `src/styles/reset.css`, CONTEXT.md D-08, CLAUDE.md §Typography.

## Color

All values from `src/styles/reset.css` — DEVLIOT grayscale palette, no changes for Phase 3.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-surface` | `#ffffff` | Article page background, Mermaid diagram background |
| Secondary (30%) | `--color-surface-alt` | `#f8f9fa` | Code block background, `<figure>` background (optional subtle tint) |
| Accent (10%) | `--color-accent` | `#333333` | Anchor `#` symbol on heading hover, copy button icon, figure caption "Figure N:" label |
| Text primary | `--color-text` | `#1a1a1a` | All prose, code text |
| Text muted | `--color-text-muted` | `#666666` | Language badge text, "Copied!" feedback text, figcaption prose |
| Destructive | — | `#cc0000` | Render error state message (fallback display only, no interactive destructive actions) |

Source: CONTEXT.md (monochrome palette, D-05 GitHub Light theme), MEMORY.md (monochrome palette, no colored accent).

### Color Exceptions

- **Shiki GitHub Light theme:** Syntax token colors (keyword blue, string green, etc.) are intentional departures from strict monochrome. They are scoped exclusively to `devliot-code` shadow DOM output. This is approved per CONTEXT.md specifics: "intentional departure from strict monochrome for readability."
- **Chart.js charts:** Must use a constrained grayscale palette within `devliot-chart`. No color outside `#1a1a1a`, `#333333`, `#666666`, `#999999`, `#cccccc`. Claude's discretion per CONTEXT.md.
- **KaTeX math:** KaTeX default CSS produces black text on transparent — no override needed.
- **Mermaid diagrams:** Must initialize with a light theme (`theme: 'default'` or `theme: 'neutral'`) compatible with white backgrounds. Avoid `theme: 'dark'`.

Accent reserved for: anchor `#` symbol (heading hover), clipboard icon on copy button, "Figure N:" figure number prefix. Never used for interactive states (hover/focus use opacity or border changes, not color change).

## Component Interaction Contracts

### `devliot-code` — Syntax-Highlighted Code Block

| State | Visual Behavior |
|-------|----------------|
| Default | Code block with `--color-surface-alt` background, Fira Code 14px, line numbers left column, language badge top-right |
| Hover | Copy button appears in top-right corner (opacity 0 → 1), clipboard SVG icon, 44×44px touch target |
| Copy click | Button shows "Copied!" text label (replaces icon), `--color-text-muted` color, returns to icon after 2000ms |
| Copy error | Button shows "Error" text label (same treatment), silent fail — no alert |

Source: CONTEXT.md D-06, D-07, D-08.

### `devliot-math` — KaTeX Formula

| State | Visual Behavior |
|-------|----------------|
| Inline (default) | Rendered inline with surrounding prose, baseline-aligned |
| Block (`display` attr) | Centered block, `--space-xl` (32px) top/bottom margin |
| Render error | Red error message from KaTeX, kept in place — do not hide |

Source: CONTEXT.md D-09.

### `devliot-diagram` — Mermaid Diagram (lazy)

| State | Visual Behavior |
|-------|----------------|
| Before viewport | Empty block, no placeholder visible |
| Loading (in viewport, importing) | No spinner — diagram appears when ready (avoid layout shift) |
| Rendered | SVG fills 100% container width, max-width: 100%, centered |
| Syntax error | Mermaid error block rendered as-is (Mermaid's built-in error display) |
| Container | Subtle border `1px solid #e0e0e0`, `--space-md` (16px) internal padding, `--space-xl` (32px) top/bottom margin |

Source: CONTEXT.md D-10, RESEARCH.md §Pattern notes (light DOM render required).

### `devliot-chart` — Chart.js Chart (lazy)

| State | Visual Behavior |
|-------|----------------|
| Before viewport | Empty canvas placeholder, no visible indicator |
| Loading | Canvas instantiates when IntersectionObserver triggers |
| Rendered | Canvas fills container, max-width: 100%, `--space-xl` margin top/bottom |
| Config error | `console.error` only — no visible error UI in Phase 3 |

Source: CONTEXT.md D-10, D-11.

### Heading Anchor Links

| State | Visual Behavior |
|-------|----------------|
| Default | No visible `#` symbol |
| Heading hover | `#` symbol appears to the left of heading text, `--color-accent` (`#333333`), cursor: pointer |
| Anchor click | URL hash updated, anchor URL copied to clipboard, no toast/feedback |

Source: CONTEXT.md D-12.

### `<figure>` / `<figcaption>` — Image with Caption

| Element | Visual Behavior |
|---------|----------------|
| `<figure>` | `--space-xl` (32px) top/bottom margin, centered block |
| `<img>` | `max-width: 100%`, `height: auto`, `border-radius: 4px` |
| `<figcaption>` | 14px, `--color-text-muted`, `--space-sm` (8px) top margin, CSS counter: "Figure N: " prefix in `--color-accent` at weight 600 |

Source: CONTEXT.md D-13, REQUIREMENTS.md ART-04.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Copy button default | _(icon only — no text label)_ |
| Copy button success | "Copied!" |
| Copy button error | "Error" |
| Figure caption prefix | "Figure N:" (auto-numbered via CSS counter) |
| Heading anchor hover | "#" (single character, no tooltip) |
| Diagram loading | _(no copy — silent load)_ |
| Render error (math) | KaTeX native error message — not overridden |
| Render error (diagram) | Mermaid native error block — not overridden |
| Article not found | "Article not found." (plain text, centered in article container) |
| Article load failed | "Could not load article. Check your connection and try again." |

Notes:
- There are no primary CTAs in this phase — Phase 3 is a pure reading/rendering phase with no user-initiated actions beyond copying.
- There are no destructive actions in this phase.
- Empty state for article body: only the "Article not found." message above. The article list (home page) is Phase 4.

Source: CONTEXT.md decisions + defaults derived from phase scope.

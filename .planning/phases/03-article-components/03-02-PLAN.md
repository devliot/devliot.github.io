Element name: `devliot-code`. Properties:
- `@property({ type: String }) lang = 'text'` — programming language for Shiki
- `@state() private _highlightedHtml = ''` — Shiki output HTML
- `@state() private _copied = false` — copy button feedback state
- `@state() private _code = ''` — raw code text extracted from textContent

**Authoring API in article HTML (per D-02):**
```html
<devliot-code lang="typescript">
const x: number = 42;
console.log(x);
</devliot-code>
```

**Lifecycle:**
- `connectedCallback()`: Call `super.connectedCallback()`. Capture `this.textContent?.trim() ?? ''` into `this._code` BEFORE Lit renders (textContent is the raw code the author wrote between the tags). Then clear `this.textContent = ''` to avoid showing raw text.
- `firstUpdated()`: Call `this._highlight()`.
- `async _highlight()`: Import `codeToHtml` from `'shiki'` and call:
  ```typescript
  const result = await codeToHtml(this._code, {
    lang: this.lang || 'text',
    theme: 'github-light',
  });
  this._highlightedHtml = result;
  ```

**Copy button logic:**
- `async _copyCode()`: Call `navigator.clipboard.writeText(this._code)`. On success, set `this._copied = true`, then `setTimeout(() => { this._copied = false }, 2000)`. On error, set a temporary "Copy failed" state (same 2s timeout).
- The button element: `<button class="copy-btn" aria-label="Copy code" @click=${this._copyCode}>`. When `_copied` is false, show clipboard SVG icon. When true, show text "Copied!" in `--color-text-muted`.

**Language badge:** A `<span class="lang-badge">` positioned absolute top-right showing `this.lang.toUpperCase()`. Always visible (not hover-dependent).

**Render method:**
```typescript
render() {
  return html`
    <div class="code-block">
      <span class="lang-badge">${this.lang.toUpperCase()}</span>
      <button class="copy-btn" aria-label="Copy code" @click=${this._copyCode}>
        ${this._copied
          ? html`<span class="copy-feedback">Copied!</span>`
          : html`<svg class="copy-icon" ...><!-- clipboard icon --></svg>`
        }
      </button>
      ${this._highlightedHtml
        ? unsafeHTML(this._highlightedHtml)
        : html`<pre><code>${this._code}</code></pre>`
      }
    </div>
  `;
}
```

**Clipboard SVG icon** (inline SVG, no icon library per UI-SPEC):
```html
<svg width="18" height="18" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
```

**A. Create `src/components/devliot-math.ts`:**

```typescript
import { LitElement, html, unsafeCSS } from 'lit';
import { customElement, state } from 'lit/decorators.js';
import { unsafeHTML } from 'lit/directives/unsafe-html.js';
import katex from 'katex';
import styles from '../styles/devliot-math.css?inline';
```

Element name: `devliot-math`. No `@property` for display — use `hasAttribute('display')` instead, since it is a boolean attribute.

State:
- `@state() private _rendered = ''` — KaTeX HTML output
- Private field: `private _latex = ''` — raw LaTeX from textContent

**Authoring API (per D-09):**
```html
<!-- Inline math -->
<devliot-math>E = mc^2</devliot-math>

<!-- Block/display math -->
<devliot-math display>\int_0^\infty e^{-x} dx = 1</devliot-math>
```

**Lifecycle:**
- `connectedCallback()`: Call `super.connectedCallback()`. Capture `this._latex = this.textContent?.trim() ?? ''`. Clear `this.textContent = ''`.
- `firstUpdated()`: Call `this._renderMath()`.
- `_renderMath()`: 
  ```typescript
  try {
    this._rendered = katex.renderToString(this._latex, {
      throwOnError: false,
      displayMode: this.hasAttribute('display'),
    });
  } catch (e) {
    this._rendered = `<span class="math-error">${(e as Error).message}</span>`;
  }
  ```

Note: `throwOnError: false` makes KaTeX render error messages inline (red text) rather than throwing. The catch block is a safety net for truly unexpected errors.

**Render method:**
```typescript
render() {
  return html`${unsafeHTML(this._rendered)}`;
}
```

Static styles: `unsafeCSS(styles)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| textContent → Shiki codeToHtml | Author-controlled code content passed to Shiki for highlighting |
| textContent → KaTeX renderToString | Author-controlled LaTeX passed to KaTeX for rendering |
| codeToHtml output → unsafeHTML | Shiki HTML output injected into shadow DOM |
| renderToString output → unsafeHTML | KaTeX HTML output injected into shadow DOM |
| user click → clipboard | Code block content written to clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | devliot-code unsafeHTML(Shiki output) | accept | Shiki output is generated from a known library processing author-controlled input. Not user-supplied. Same trust as any build-time code transformation. |
| T-03-05 | Tampering | devliot-math unsafeHTML(KaTeX output) | accept | KaTeX output is generated from a known library processing author-controlled LaTeX. No user input path exists. |
| T-03-06 | Denial of Service | devliot-math with complex LaTeX | accept | KaTeX has built-in limits on nesting depth and macro expansion. No user-facing input — only author writes LaTeX. |
| T-03-07 | Information Disclosure | clipboard writeText(code) | accept | Only author-written code is copied. No secrets in article code blocks. |
</threat_model>

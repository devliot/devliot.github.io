**A. Rewrite `src/pages/devliot-article-page.ts`:**

Replace the current placeholder with a generic HTML renderer. Key implementation:

```typescript
import { LitElement, html, unsafeCSS } from 'lit';
import { customElement, property, state } from 'lit/decorators.js';
import { unsafeHTML } from 'lit/directives/unsafe-html.js';
import articleStyles from '../styles/article.css?inline';
import codeStyles from '../styles/code.css?inline';
```

Properties and state:
- `@property({ type: String }) slug = ''` — set by router
- `@state() private _html = ''` — fetched article HTML
- `@state() private _error = ''` — error message string

Lifecycle:
- `willUpdate(changed)`: When `slug` changes and is non-empty, call `this._loadArticle()`.
- `async _loadArticle()`: Fetch from `${import.meta.env.BASE_URL}articles/${this.slug}.html`. If response is OK, set `this._html = await res.text()` and clear `_error`. If 404, set `_error = 'Article not found.'`. If network error, set `_error = 'Could not load article. Check your connection and try again.'` (per UI-SPEC copywriting contract).
- `updated(changed)`: When `_html` changes AND is non-empty, call `this._injectHeadingAnchors()` after a microtask (`await this.updateComplete`).

Heading anchor injection (`_injectHeadingAnchors()`):
- Query `this.shadowRoot.querySelector('article')`.
- `querySelectorAll('h2, h3, h4, h5, h6')` — NOT h1 (the article title).
- For each heading: generate an `id` from `heading.textContent.toLowerCase().replace(/\s+/g, '-').replace(/[^\w-]/g, '')`.
- Set `heading.id = id`.
- Create an `<a>` element with `className = 'heading-anchor'`, `href = '#'`, `textContent = '#'`, `setAttribute('aria-hidden', 'true')`.
- On click: `e.preventDefault()`, copy `${window.location.origin}${window.location.pathname}${window.location.hash.split('?')[0]}?section=${id}` to clipboard via `navigator.clipboard.writeText()`, then `heading.scrollIntoView({ behavior: 'smooth' })`.
- `heading.prepend(anchor)`.
- Also in `connectedCallback()`: check `window.location.hash` or `window.location.href` for a `?section=` query param, and if present, scroll to that heading ID after content loads.

Render method:
```typescript
render() {
  if (this._error) {
    return html`<article class="error-state"><p>${this._error}</p></article>`;
  }
  return html`<article>${unsafeHTML(this._html)}</article>`;
}
```

Static styles: combine articleStyles and codeStyles via array:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fetch → unsafeHTML | HTML fetched from local static files is injected into DOM without sanitization |
| clipboard API | Writing to user clipboard on heading anchor click |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | devliot-article-page unsafeHTML | accept | Article HTML is author-controlled (committed to repo), not user-supplied. No DOMPurify needed — same trust model as the app's own templates. If articles ever accept user input, revisit with DOMPurify. |
| T-03-02 | Spoofing | fetch URL path | mitigate | Validate slug contains only alphanumeric, hyphens, and underscores before constructing fetch URL. Reject slugs with `..`, `/`, or special characters to prevent path traversal. |
| T-03-03 | Information Disclosure | clipboard writeText | accept | Only the anchor URL (public page URL) is copied. No sensitive data. Clipboard API requires secure context (HTTPS or localhost). |
</threat_model>

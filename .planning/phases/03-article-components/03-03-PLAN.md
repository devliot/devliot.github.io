<objective>
Build the `devliot-diagram` component (Mermaid diagrams, lazy-loaded, light DOM rendering) and the `devliot-chart` component (Chart.js charts, lazy-loaded). Both use IntersectionObserver to defer heavy library imports until the element enters the viewport.

Purpose: Mermaid (~2MB) and Chart.js (~200KB) are heavy libraries that most articles will not use. Lazy-loading ensures fast initial page loads. Mermaid requires a critical workaround: render in light DOM to bypass Shadow DOM incompatibility (mermaid#6306).

Output: Two registered custom elements (`<devliot-diagram>` and `<devliot-chart>`) that lazy-load their respective libraries on viewport entry.
</objective>

**CRITICAL per RESEARCH.md Pitfall 1:** Mermaid is incompatible with Shadow DOM (github.com/mermaid-js/mermaid/issues/6306). Override `createRenderRoot()` to return `this` (light DOM rendering).

Element name: `devliot-diagram`.

Private fields:
- `private _definition = ''` — Mermaid definition text from textContent
- `private _observer?: IntersectionObserver` — viewport observer
- `private _rendered = false` — prevent double rendering
- `@state() private _error = ''` — error message if Mermaid fails
- Static counter: `private static _counter = 0` — unique ID for each diagram

**CRITICAL override:**
```typescript
override createRenderRoot() {
  return this; // Light DOM — required for Mermaid compatibility
}
```

**Lifecycle:**
- `connectedCallback()`: Call `super.connectedCallback()`. Capture `this._definition = this.textContent?.trim() ?? ''`. Clear `this.textContent = ''`. Create the IntersectionObserver:
  ```typescript
  this._observer = new IntersectionObserver(
    async (entries) => {
      if (entries[0].isIntersecting && !this._rendered) {
        this._observer!.disconnect();
        await this._renderDiagram();
      }
    },
    { rootMargin: '200px' }
  );
  this._observer.observe(this);
  ```
  Note: Since we use light DOM, `firstUpdated` may not fire as expected for LitElement without shadow DOM. Observe directly in `connectedCallback()` after a microtask: `queueMicrotask(() => this._observer?.observe(this))`.

- `async _renderDiagram()`:
  ```typescript
  try {
    const { default: mermaid } = await import('mermaid');
    mermaid.initialize({ startOnLoad: false, theme: 'neutral' });
    const id = `mermaid-${DevliotDiagram._counter++}`;
    const { svg, bindFunctions } = await mermaid.render(id, this._definition);
    // Create container div for the SVG
    const container = document.createElement('div');
    container.className = 'diagram-svg-container';
    container.innerHTML = svg;
    // Clear any existing content and append
    this.innerHTML = '';
    this.appendChild(container);
    if (bindFunctions) bindFunctions(container);
    this._rendered = true;
  } catch (e) {
    // Mermaid syntax error — show error per UI-SPEC (native Mermaid error block)
    console.error('Mermaid render error:', e);
    this._error = (e as Error).message;
    this.innerHTML = `<div class="diagram-error">${this._error}</div>`;
  }
  ```

- `disconnectedCallback()`: Call `super.disconnectedCallback()`. `this._observer?.disconnect()`.

**Render method** (used before lazy load triggers — shows empty container):
```typescript
render() {
  // Light DOM: this method adds structure before the diagram loads
  return html``;
}
```

Since we're in light DOM mode and mutating innerHTML directly in `_renderDiagram`, the render method can return empty. The initial state is the empty element before IntersectionObserver fires.

1. Add a private property `private currentQuery = new URLSearchParams()` to the `HashRouter` class.

2. Replace the `_onHashChange` method body with logic that splits the raw hash on `?` before matching routes:
   ```typescript
   private _onHashChange = () => {
     const hash = window.location.hash;
     const raw = hash ? hash.slice(1) || '/' : '/';
     const qIdx = raw.indexOf('?');
     const pathPart = qIdx === -1 ? raw : raw.slice(0, qIdx);
     const queryPart = qIdx === -1 ? '' : raw.slice(qIdx + 1);
     this.currentPath = pathPart.length > 1 && pathPart.endsWith('/')
       ? pathPart.slice(0, -1) : (pathPart || '/');
     this.currentQuery = new URLSearchParams(queryPart);
     this.host.requestUpdate();
   };
   ```

3. Add a public method to expose parsed query params:
   ```typescript
   getQuery(): URLSearchParams {
     return this.currentQuery;
   }
   ```

The existing route patterns (`/` and `/article/:slug`) must continue to work because `_match` now receives only the path portion (before `?`), not the full hash.

   const registry = JSON.parse(readFileSync('public/articles/index.json', 'utf8'));

       category: article.category,
       tags: article.tags,

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Filter chip inner horizontal padding supplement, tag gap |
| sm | 8px | Filter chip vertical padding, tag gap within article row |
| md | 16px | Section horizontal padding, article row padding-inline |
| lg | 24px | Gap between hero and filter chips, section padding |
| xl | 32px | Gap between filter strip and article list |
| 2xl | 48px | Hero section vertical padding (mobile) |
| 3xl | 64px | Hero section vertical padding (desktop) |

Exceptions:
- Search input touch target: 44px minimum height (accessibility, WCAG 2.5.5)
- Filter chip height: 28px (compact density per D-01 decision)
- Article list row height: not fixed — single-line layout with natural padding of 8px top/bottom

Source: existing `--space-*` tokens in `reset.css`, no new tokens needed.

**Accent reserved for:**
- Active filter chip: background `--color-accent` (#333333), text `--color-surface` (#ffffff)

## Component Contracts

### 1. Filter Chip Strip (new component: `devliot-filter-chips` or inline in home page)

**Location:** Home page, between hero section and article list. Horizontally scrollable on mobile.

**Anatomy:**
```
[ All ] [ IA ] [ Java ] [ Tutorial ] [ demo ] [ components ]
```
- One chip per unique tag derived from `articles[].tags` + `articles[].category` merged and deduplicated
- "All" chip always first; remaining chips in alphabetical order
- Chips rendered as `<button>` elements

**States:**
| State | Background | Border | Text | Cursor |
|-------|-----------|--------|------|--------|
| Default | `--color-surface-alt` (#f8f9fa) | 1px solid transparent | `--color-text` (#1a1a1a) | pointer |
| Hover (inactive) | #eeeeee (midpoint between surface-alt and accent) | 1px solid transparent | `--color-text` | pointer |
| Active | `--color-accent` (#333333) | 1px solid #333333 | `--color-surface` (#ffffff) | default |
| Focus-visible | any | 2px solid `--color-accent`, offset 2px | — | — |

**Dimensions:**
- Height: 28px
- Padding: 4px 12px (xs vertical, 12px horizontal — exception to 8-point scale for chip density)
- Border-radius: 4px
- Gap between chips: 8px (`--space-sm`)
- Font: 14px / weight 400 (default), 14px / weight 600 (active)

**Layout:**
- Strip container: `display: flex; flex-wrap: wrap; gap: 8px; padding: 16px 0 24px`
- On mobile: allow wrap to second line if needed (no horizontal scroll — wrap is cleaner at narrow width)

**Behavior:**
- Clicking an active chip deactivates it (returns to "All")
- Clicking "All" deactivates any active filter
- Only one chip active at a time (single-select)
- Tag filter state communicated via URL hash query (e.g., `/#/?tag=Java`) so links are shareable and back-button works

### 2. Article List Row (new component or template in home page)

**Location:** Home page, below filter chip strip.

**Anatomy per row (one article):**
```
2026-04-11   Tutorial   Article Components Demo
                        demo  components  reference
```

Column breakdown:
- Date: 14px, `--color-text-muted`, monospace (`--font-family-mono`), fixed-width `90px`, no wrap
- Category: 14px, `--color-text-muted`, fixed-width `80px`, no wrap, uppercase tracking (0.5px letter-spacing)
- Title: 16px, `--color-text`, flex-grow, truncate with ellipsis on overflow, hyperlink to article
- Tags row (below, indented to title column start): 14px, `--color-text-muted`, comma-separated or space-separated inline

**Layout:**
```
display: grid;
grid-template-columns: 90px 80px 1fr;
grid-template-rows: auto auto;
```
Tags span the 3rd column below the title.

**Row spacing:** 8px padding-block per row. 1px solid `#eeeeee` separator between rows (not on last row).

**States:**
| State | Effect |
|-------|--------|
| Default | as described |
| Row hover | Title color shifts to `--color-accent` (#333333); background stays white |
| Title link focus-visible | 2px solid `--color-accent` outline |

**Tag click behavior:** Clicking a tag text in the row activates that tag's filter chip (navigates to `/#/?tag={tag}`).

**Article ordering:** Newest first — sort by `date` field descending before render.

**Mobile layout (< 768px):**
- Drop date and category columns; show only title on first line, date + category on second line in muted text
- Grid collapses to single column: `grid-template-columns: 1fr`

### 3. Search Input in Sticky Header (modification to `devliot-header.ts`)

**Location:** Right side of sticky header, replacing or alongside the hamburger on desktop.

**Collapsed state (default):**
- A search icon button (magnifier, 20px, `--color-accent`)
- Minimum touch target 44x44px
- `aria-label="Search articles"`

**Expanded state (after click):**
- Icon button remains visible on the left of the input
- Text input appears to the left of the icon (slides in, width: 180px on mobile, 240px on desktop)
- Input placeholder: `"Search articles…"`
- Input: 16px, `--color-text`, background `--color-surface`, bottom-border 1px solid `--color-accent`, no box border (borderless except bottom)
- Pressing Escape or clicking outside collapses the input and clears the value
- `aria-label="Search articles"` on the input

**Expand/collapse animation:**
- Width transition: `width 0.2s ease` from 0 to target width
- Do NOT use opacity-only fade (reduces perceived snap)

**Header layout with search expanded (desktop ≥ 1280px):**
- Logo (left) | [search input ← search icon] (right)
- Hamburger hidden at desktop

**Header layout with search expanded (mobile/tablet < 1280px):**
- Logo (left) | [search icon] + [hamburger] (right)
- When search is expanded on mobile, the input overlays the header row below the logo row (full width, 44px tall, bottom-border style)

**Results behavior:**
- Search filters the article list on the home page in real time (debounce: 200ms)
- If not on home page, search navigates to `/#/` with query `?q={term}` so the home page reads and applies it
- No dropdown/popover — results ARE the filtered article list

### 4. Empty State

**Location:** Article list area on home page, when filter or search yields zero results.

**Copy:** `No articles found.`

**Visual treatment:**
- Centered in the article list container
- 64px padding-block (`--space-3xl`)
- 16px, weight 400, color `--color-text-muted`
- No icon, no illustration — text only (matches existing `.error-state` pattern in article.css)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click filter chip (inactive) | Activates chip, updates URL hash to `/#/?tag={tag}`, filters list |
| Click filter chip (active) | Deactivates chip, updates URL hash to `/#/`, shows all articles |
| Click "All" chip | Deactivates any active tag, shows all articles |
| Click tag in article row | Activates corresponding filter chip (same as clicking the chip) |
| Type in search | 200ms debounce, then filters list client-side via FlexSearch |
| Press Escape in search | Clears and collapses search input |
| Click outside search input | Collapses search input (if empty) |
| Navigate to `/#/?tag=X` directly | Home page reads URL param, activates chip X on mount |
| Navigate to `/#/?q=term` directly | Home page reads URL param, populates search and filters on mount |
| Simultaneous tag + search | Both filters apply (AND logic: title/body/tags match search AND tag matches) |

## Accessibility Contract

- All filter chips: `<button>` with visible focus ring (2px solid `--color-accent`)
- Active chip: `aria-pressed="true"`, inactive: `aria-pressed="false"`
- Filter strip: `role="group"`, `aria-label="Filter by tag"`
- Search button: `aria-label="Search articles"`, `aria-expanded="false/true"`
- Search input: `role="search"`, `aria-label="Search articles"`
- Article list: `<ul>` with `<li>` per article, `role` not overridden
- Empty state: announced to screen readers via `aria-live="polite"` on the list container

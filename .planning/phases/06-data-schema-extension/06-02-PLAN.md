<objective>
Wire the two TypeScript consumer files to use the centralised Article types from Plan 01, replacing inline interfaces and adding explicit type annotations at `.json()` deserialization boundaries.

Purpose: Without these annotations, `response.json()` returns `any` (because `tsconfig.json` has `strict: false`), making the success criteria "produces no TypeScript errors" trivially true regardless of schema correctness. Annotating the call sites makes `tsc` actually enforce the Article schema — a typo in a field name now produces a compile error. This is the "type enforcement" half of Phase 6.

Output: Updated `devliot-home-page.ts` and `devliot-article-page.ts` with type imports + annotations. Full build + Playwright regression green.
</objective>

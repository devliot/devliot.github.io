Modify `src/devliot-app.ts` to add a ResizeObserver that publishes the header's pixel height as a CSS custom property on `document.documentElement`.

1. Add a private field: `private _headerObserver?: ResizeObserver;`

2. Add `firstUpdated()` lifecycle method (the class does not currently have one):
```typescript
firstUpdated() {
  const header = this.renderRoot.querySelector('devliot-header');
  if (header) {
    this._headerObserver = new ResizeObserver(([entry]) => {
      const height = entry.borderBoxSize?.[0]?.blockSize
        ?? (entry.target as HTMLElement).offsetHeight;
      document.documentElement.style.setProperty(
        '--header-height', `${height}px`
      );
    });
    this._headerObserver.observe(header);
  }
}
```

3. Add `disconnectedCallback()` for cleanup:
```typescript
disconnectedCallback() {
  super.disconnectedCallback();
  this._headerObserver?.disconnect();
}
```

Per D-09: the ResizeObserver goes in `devliot-app.ts` because it lives for the full SPA session (never unmounts), renders `<devliot-header>`, and publishes globally to `:root`. Phase 8 header content changes will be absorbed automatically since ResizeObserver fires on element dimension changes.

PITFALL (RESEARCH.md Pitfall 5): Access `entry.borderBoxSize[0].blockSize` (array, not direct property). Fall back to `offsetHeight` if borderBoxSize is undefined.

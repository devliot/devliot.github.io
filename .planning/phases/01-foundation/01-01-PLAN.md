5. Create `tsconfig.json` in the project root with the exact content from RESEARCH.md Pattern 1 (official lit-ts template config):
   ```json
   {
     "compilerOptions": {
       "target": "es2023",
       "experimentalDecorators": true,
       "useDefineForClassFields": false,
       "module": "esnext",
       "lib": ["ES2023", "DOM"],
       "types": ["vite/client"],
       "skipLibCheck": true,
       "moduleResolution": "bundler",
       "allowImportingTsExtensions": true,
       "verbatimModuleSyntax": true,
       "moduleDetection": "force",
       "noEmit": true,
       "noUnusedLocals": true,
       "noUnusedParameters": true,
       "erasableSyntaxOnly": true,
       "noFallthroughCasesInSwitch": true
     },
     "include": ["src"]
   }
   ```
   CRITICAL: `useDefineForClassFields: false` and `experimentalDecorators: true` are required or Lit `@property()` decorators will silently break (RESEARCH.md Pitfall 2).

  <action>
1. Create `vite.config.ts` in the project root:
   ```typescript
   import { defineConfig } from 'vite';

   export default defineConfig({
     base: '/',
     build: {
       outDir: 'dist',
       target: 'es2023',
     },
   });
   ```
   `base: '/'` per D-12 (root base path for GitHub Pages).

2. Create `index.html` in the project root:
   ```html
   <!DOCTYPE html>
   <html lang="fr">
     <head>
       <meta charset="UTF-8" />
       <meta name="viewport" content="width=device-width, initial-scale=1.0" />
       <title>DEVLIOT</title>
       <link rel="stylesheet" href="/src/styles/reset.css" />
     </head>
     <body>
       <devliot-app></devliot-app>
       <script type="module" src="/src/main.ts"></script>
     </body>
   </html>
   ```
   Note: `lang="fr"` per UI-SPEC accessibility baseline (site is French). The reset.css is loaded globally via `<link>` (not Shadow DOM scoped) because it contains `:root` custom properties that all components need to inherit.

3. Create `src/main.ts`:
   ```typescript
   import './devliot-app.js';
   ```
   Single side-effect import that registers the `<devliot-app>` custom element. The `.js` extension is required for TypeScript with `verbatimModuleSyntax` — TypeScript does not rewrite import paths, and Vite resolves `.js` to `.ts` at dev time.

4. Create `src/styles/reset.css` with the global reset AND all Phase 1 design tokens from UI-SPEC:
   ```css
   *,
   *::before,
   *::after {
     box-sizing: border-box;
     margin: 0;
     padding: 0;
   }

   :root {
     /* Colors — Phase 1 skeleton, Phase 2 overrides */
     --color-surface: #ffffff;
     --color-surface-alt: #f5f5f5;
     --color-accent: #0000ee;

     /* Spacing scale — multiples of 4 */
     --space-xs: 4px;
     --space-sm: 8px;
     --space-md: 16px;
     --space-lg: 24px;
     --space-xl: 32px;
     --space-2xl: 48px;
     --space-3xl: 64px;

     /* Typography — skeleton, Phase 2 overrides */
     --font-family: system-ui, -apple-system, sans-serif;
     --font-size-body: 16px;
     --font-size-label: 14px;
     --font-size-heading: 28px;
     --font-weight-regular: 400;
     --font-weight-semibold: 600;
     --line-height-body: 1.5;
     --line-height-heading: 1.2;
   }

   body {
     font-family: var(--font-family);
     font-size: var(--font-size-body);
     line-height: var(--line-height-body);
     color: #1a1a1a;
     background-color: var(--color-surface);
   }
   ```

5. Create a minimal `src/devliot-app.ts` stub so that `npm run build` passes (the real implementation comes in Plan 02):
   ```typescript
   import { LitElement, html } from 'lit';
   import { customElement } from 'lit/decorators.js';

   @customElement('devliot-app')
   export class DevliotApp extends LitElement {
     render() {
       return html`<main><p>DEVLIOT loading...</p></main>`;
     }
   }
   ```
   This stub exists solely to make `tsc && vite build` pass. Plan 02 replaces the entire body of this file with the real app shell (header + router outlet + footer).

6. Run `npm run build` to verify zero TypeScript errors and dist/ output is generated.

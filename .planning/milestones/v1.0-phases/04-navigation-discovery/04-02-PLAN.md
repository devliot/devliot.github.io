```typescript
private _onHashChange = () => {
  const hash = window.location.hash.slice(1) || '/';
  const qIdx = hash.indexOf('?');
  if (qIdx !== -1) {
    const params = new URLSearchParams(hash.slice(qIdx + 1));
    this._activeTag = params.get('tag');
```

**6. Compute derived values in render (or as getters):**
- `_allTags`: Deduplicated set of all tags from all articles, combining each article's `tags` array AND `category` field. Sorted alphabetically. Computed from `_articles`.
  ```typescript
  private get _allTags(): string[] {
    const tagSet = new Set<string>();
    for (const a of this._articles) {
      if (a.category) tagSet.add(a.category);
      for (const t of a.tags) tagSet.add(t);
    }
    return [...tagSet].sort((a, b) => a.localeCompare(b));
  }
  ```

## Trust Boundaries

|----------|-------------|
| index.json -> article list render | Article metadata rendered via Lit template auto-escaping |
| URL hash tag param -> filter predicate | User-controlled tag value used as array filter predicate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | Tag filter from URL | accept | Tag value from `?tag=X` is compared against pre-loaded article tags array via strict equality — no injection vector. Lit template auto-escapes all interpolated values. |
| T-04-05 | Spoofing | Article title XSS via unsafeHTML | accept | Article HTML is authored by the site owner (not user-generated), already rendered via unsafeHTML in Phase 3. No change in trust model. |
| T-04-06 | Tampering | Tag link buttons in article page | accept | Tags fetched from index.json (author-controlled), rendered via Lit auto-escaping `${tag}` — no unsafeHTML for tag text. |
</threat_model>

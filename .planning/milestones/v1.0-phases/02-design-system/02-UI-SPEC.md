# Phase 2 — UI Design Contract

> Visual and interaction contract for the Design System phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual CSS custom properties in `src/styles/reset.css` |
| Preset | not applicable |
| Component library | none — Lit.js web components (non-negotiable per CLAUDE.md) |
| Icon library | none (Phase 2 scope: brand + layout only) |
| Font (body) | Inter — self-hosted, no CDN |
| Font (code) | Fira Code — self-hosted, no CDN |

**Rationale:** shadcn is a React component library and is incompatible with this Lit.js project. Design tokens are declared as CSS custom properties in `:root` within `src/styles/reset.css`, which pierce Shadow DOM on all Lit components.

## Spacing Scale

Source: Phase 1 — established in `src/styles/reset.css`, Phase 2 carries forward unchanged.

| Token | CSS Custom Property | Value | Usage |
|-------|--------------------|----|-------|
| xs | `--space-xs` | 4px | Icon gaps, inline padding |
| sm | `--space-sm` | 8px | Compact element spacing |
| md | `--space-md` | 16px | Default element spacing, header/footer padding |
| lg | `--space-lg` | 24px | Section padding, hero padding |
| xl | `--space-xl` | 32px | Layout gaps |
| 2xl | `--space-2xl` | 48px | Major section breaks |
| 3xl | `--space-3xl` | 64px | Page-level spacing |

Exceptions:
- Content column max-width: 720px (not a spacing token — a layout constraint)
- Mobile sticky header height: use `--space-3xl` (64px) as minimum touch target clearance for content below header

## Typography

Source: D-06, D-07, D-08 from `02-CONTEXT.md`. Sizes and weights carry forward from Phase 1 skeleton; font families are the Phase 2 change.

| Role | CSS Custom Property | Size | Weight | Line Height | Font |
|------|--------------------|----|--------|-------------|------|
| Body | `--font-size-body` | 16px | 400 (regular) | 1.5 | Inter |
| Label | `--font-size-label` | 14px | 400 (regular) | 1.5 | Inter |
| Heading | `--font-size-heading` | 28px | 600 (semibold) | 1.2 | Inter |
| Display (ASCII logo) | not a CSS text var — use `font-size` directly | 10px–14px depending on breakpoint | 700 (bold) | 1.0 | Fira Code (monospace, preserves ASCII art alignment) |

**Font weight tokens:**
- `--font-weight-regular: 400` — body text, labels, captions
- `--font-weight-semibold: 600` — headings, logo link in header

**ASCII logo specifics (D-01, D-02, D-03):**
- Rendered as `<pre>` element using Fira Code at 14px desktop / 10px mobile
- Line height 1.0 to preserve block letter proportions
- Scales via `font-size` reduction only — never `transform: scale()` or plain-text fallback
- Full-size version in home page hero section
- Reduced-size version in site header

**Font loading:**
- `@font-face` declarations in `src/styles/fonts.css` (new file — Phase 2 creates it)
- Loaded via `<link rel="stylesheet" href="/src/styles/fonts.css">` in `index.html`
- Formats: woff2 (primary) + woff (fallback)
- `font-display: swap` on all declarations

## Color

Source: D-04, D-05 from `02-CONTEXT.md`. Phase 2 replaces Phase 1 skeleton values.

| Role | CSS Custom Property | Value | Usage |
|------|--------------------|----|-------|
| Dominant (60%) | `--color-surface` | `#ffffff` | Page background, article body surface |
| Secondary (30%) | `--color-surface-alt` | `#f8f9fa` | Header background, footer background, code block background |
| Accent (10%) | `--color-accent` | `#0077b6` | See reserved list below |
| Text primary | `--color-text` | `#1a1a1a` | All body text, headings |
| Text muted | `--color-text-muted` | `#666666` | Footer text, captions, labels |
| Destructive | not in Phase 2 scope | `#dc2626` | Not applicable — no destructive actions in this phase |

**Accent reserved for (explicit list):**
1. Header logo link (hover state only — default state uses `--color-text`)
2. Hyperlinks within article body (future Phase 3 — declared now for consistency)
3. Active/focus ring on interactive elements (2px solid `#0077b6`)

Accent is NOT used for: backgrounds, borders, text color on headers or footers, decorative elements.

**Phase 1 override required:** `reset.css` currently has `--color-accent: #0000ee` and `--color-surface-alt: #f5f5f5`. Phase 2 must update both to the values above.

## Responsive Layout Contract

Source: INFRA-05, D-09, D-10, D-11 from `02-CONTEXT.md`.

| Breakpoint | Name | Min-width | Content max-width | Behavior |
|------------|------|-----------|-------------------|----------|
| Mobile | `mobile` | 375px | 100% (full width) | Single column, padded 16px each side |
| Tablet | `tablet` | 768px | 100% (full width) | Single column, padded 24px each side |
| Desktop | `desktop` | 1280px | 720px centered | Centered content column, auto margins |

**CSS breakpoint tokens (new in Phase 2):**
```css
--breakpoint-tablet: 768px;
--breakpoint-desktop: 1280px;
```

**Header behavior (D-10):**
- `position: sticky; top: 0; z-index: 100` on all breakpoints
- Contains hamburger menu placeholder (button element, no functionality — Phase 4 activates it)
- Hamburger visible on mobile and tablet; hidden on desktop

**Code block behavior (D-11):**
- `overflow-x: auto; white-space: pre` — horizontal scroll, no line wrapping
- Applied to all `<pre>` and `<code>` elements globally in `reset.css`

**Content max-width pattern:**
```css
.content-column {
  max-width: 720px;
  margin-inline: auto;
  padding-inline: var(--space-md);
}
```
Applied to `<main>` in `src/styles/app.css` at desktop breakpoint.

## ASCII Logo Contract

Source: D-01, D-02, D-03 from `02-CONTEXT.md`. OPENCODE-inspired block letter style.

**Character style:** Full-width block letters using `#` or `█` characters. Bold, terminal aesthetic. Each letter is approximately 5 characters tall.

**Example structure (to be generated by executor):**
```
██████╗ ███████╗██╗   ██╗██╗     ██╗ ██████╗ ████████╗
██╔══██╗██╔════╝██║   ██║██║     ██║██╔═══██╗╚══██╔══╝
██║  ██║█████╗  ██║   ██║██║     ██║██║   ██║   ██║
██║  ██║██╔══╝  ╚██╗ ██╔╝██║     ██║██║   ██║   ██║
██████╔╝███████╗ ╚████╔╝ ███████╗██║╚██████╔╝   ██║
╚═════╝ ╚══════╝  ╚═══╝  ╚══════╝╚═╝ ╚═════╝    ╚═╝
```

**Placement:**
- Hero (`src/styles/home.css`): full-size, 14px Fira Code, centered
- Header (`src/styles/header.css`): reduced, 10px Fira Code, left-aligned, inline with nav area

**Accessibility:** wrap in `<pre aria-label="DEVLIOT">` to provide screen reader text.

## Copywriting Contract

| Element | Copy |

| Header logo link `aria-label` | `DEVLIOT — accueil` |
| ASCII logo `aria-label` | `DEVLIOT` |
| Hamburger button `aria-label` | `Ouvrir le menu` |
| Footer text | `© 2026 DEVLIOT` |
| Primary CTA | Not applicable — Phase 2 has no CTA |
| Empty state | Not applicable — Phase 2 has no data-driven lists |
| Error state | Not applicable — Phase 2 has no async operations |
| Destructive confirmation | Not applicable — Phase 2 has no destructive actions |

**Language note:** The site serves a French-speaking author. Static copy uses French for labels and UI elements (`Ouvrir le menu`, not `Open menu`).

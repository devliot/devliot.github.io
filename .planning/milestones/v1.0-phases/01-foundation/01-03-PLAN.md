1. Create the directory `.github/workflows/` if it does not exist.

2. Create `.github/workflows/deploy.yml` with the following exact content:

   ```yaml
   name: Deploy to GitHub Pages

   on:
     workflow_dispatch:

   permissions:
     contents: read
     pages: write
     id-token: write

   concurrency:
     group: pages
     cancel-in-progress: false

   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: lts/*
             cache: npm
         - run: npm ci
         - run: npm run build
         - uses: actions/configure-pages@v5
         - uses: actions/upload-pages-artifact@v3
           with:
             path: dist

     deploy:
       needs: build
       runs-on: ubuntu-latest
       environment:
         name: github-pages
         url: ${{ steps.deployment.outputs.page_url }}
       steps:
         - uses: actions/deploy-pages@v4
           id: deployment
   ```

   Key details per decisions:
   - D-11: `on: workflow_dispatch` only — no `push:` or `pull_request:` triggers
   - D-13: Build step is `npm run build` only — no `npm run lint`, no `npm test`
   - D-12: Base path is `/` (configured in vite.config.ts, not in the workflow)
   - `permissions.pages: write` and `permissions.id-token: write` are required for the OIDC-based Pages deployment
   - `concurrency.cancel-in-progress: false` ensures a running deploy is not cancelled by a new trigger
   - `actions/upload-pages-artifact@v3` uploads the `dist/` directory built by Vite
   - `actions/deploy-pages@v4` deploys the artifact to GitHub Pages

   IMPORTANT per RESEARCH.md Pitfall 4: This workflow file must be pushed to the `main` branch before the "Run workflow" button appears in the GitHub Actions UI.

   IMPORTANT per RESEARCH.md Pitfall 5: The user must go to repo Settings > Pages > Source and select "GitHub Actions" (not "Deploy from a branch") before the first deploy will work.

3. Validate the YAML is syntactically correct by running:

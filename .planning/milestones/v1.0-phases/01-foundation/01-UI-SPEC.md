# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher.
> Phase 1 delivers structural skeleton only — no brand identity, no design tokens. Full design system lands in Phase 2 (BRAND-01, BRAND-02).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — Lit.js web components, handcrafted |
| Icon library | none in Phase 1 |
| Font | system-ui stack (browser default) — custom font in Phase 2 |

**Rationale:** This project uses Lit.js (non-negotiable per CLAUDE.md). shadcn is React-only and does not apply. Phase 1 installs no design tooling; Phase 2 establishes the brand. All CSS in Phase 1 is structural/reset only and must not hard-code brand values that Phase 2 will own.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing — body padding, header/footer padding |
| lg | 24px | Section padding — home hero section padding |
| xl | 32px | Layout gaps — main content area top/bottom margin |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none in Phase 1. Full spacing system arrives in Phase 2.

**Implementation note (from CONTEXT.md D-03):** Spacing values live in external CSS files imported via `?inline` + `unsafeCSS()`. Do not embed spacing in `static styles` css tagged templates. CSS custom properties are the preferred mechanism: `--space-md: 16px`.

---

## Typography

Phase 1 uses browser defaults. These are skeleton values that Phase 2 will replace. Do not hard-code font families or refined type scales — use only structural rules.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading (h1) | 28px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Font family:** `system-ui, -apple-system, sans-serif` — skeleton only, Phase 2 overrides.

**Lit implementation:** Set on `:host` in each component's external CSS file. The app shell (`devliot-app`) sets no typography; page components (`devliot-home-page`, `devliot-article-page`) own their own type rules within their shadow roots.

---

## Color

Phase 1 uses a minimal neutral palette. No brand colors. Phase 2 owns brand color decisions (BRAND-01, BRAND-02).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background, component surfaces — default browser white |
| Secondary (30%) | `#f5f5f5` | Header background, footer background — visually distinct from content |
| Accent (10%) | `#0000ee` (browser default link color) | Only: the DEVLIOT logo/title link in the header (D-10) |
| Destructive | not applicable | No destructive actions in Phase 1 |

Accent reserved for: header DEVLIOT title link only. No other interactive elements exist in Phase 1.

**Phase 2 note:** These values are placeholders. Phase 2 will define the real brand palette. Phase 1 CSS must use CSS custom properties (`--color-surface`, `--color-surface-alt`, `--color-accent`) so Phase 2 can override by redefining variables, not by patching every rule.

## Component Inventory

All components are Lit custom elements with Shadow DOM (D-04). External CSS via `?inline` + `unsafeCSS()` (D-03). All tags prefixed `devliot-` (RESEARCH.md anti-pattern #6).

### devliot-app (App Shell)

**File:** `src/devliot-app.ts`
**Responsibility:** Layout container. Renders header + main (router outlet) + footer. Owns no visual styles beyond the structural three-row layout.

**Layout:**
- Display: `flex`, `flex-direction: column`, `min-height: 100vh`
- Header: natural height (content-driven)
- Main: `flex: 1` (grows to fill remaining viewport height)
- Footer: natural height (content-driven)

**States:**
| State | Trigger | Visual |
|-------|---------|--------|
| Route matched | HashRouter finds a route | Renders matched page component in `<main>` |
| 404 | No route matches current hash | Renders inline "404 — Page not found" text in `<main>` |

### devliot-header

**File:** `src/components/devliot-header.ts`
**Responsibility:** Site header. Contains DEVLIOT logo/title as a link to `/#/` (D-10). No category nav (comes Phase 4).

**Layout:** `display: flex`, `align-items: center`, `padding: 16px` (md token). No other content.

**States:** Stateless — always renders the same.

### devliot-footer

**File:** `src/components/devliot-footer.ts`
**Responsibility:** Minimal footer. Copyright text. Discretionary content (CONTEXT.md — "Claude's Discretion: Footer content").

**Layout:** `padding: 16px` (md token), centered text.

**States:** Stateless.

### devliot-home-page

**File:** `src/pages/devliot-home-page.ts`
**Responsibility:** Home route (`/`). Placeholder hero section with intro text (D-09). Replaced by article list in Phase 4.

**Layout:** `<section class="hero">` with `padding: 24px` (lg token).

**States:** Stateless — always shows placeholder.

### devliot-article-page

**File:** `src/pages/devliot-article-page.ts`
**Responsibility:** Article route (`/article/:slug`). Stub only — displays slug name. Full content rendering comes in Phase 3.

**Layout:** `<article>` wrapper, `padding: 24px` (lg token).

**States:**
| State | Trigger | Visual |
|-------|---------|--------|
| Stub loaded | Any slug navigated to | `h1` with slug value + one paragraph of placeholder copy |

## Interaction Contract

Phase 1 has exactly one interactive element: the DEVLIOT header link.

| Interaction | Element | Behavior |
|-------------|---------|----------|
| Click DEVLIOT title | `<a href="/#/">` in devliot-header | Navigates to home route. Browser handles natively via hash change. HashRouter picks up `hashchange` event. |
| Direct URL load | Any `/#/...` URL | `hostConnected()` on HashRouter calls `_onHashChange()` immediately — resolves to correct route or 404. |
| Browser back/forward | — | `hashchange` fires, HashRouter re-evaluates route, host re-renders. |

**Focus management:** No custom focus management needed in Phase 1. All navigation is hash-based and browser-native.

**Keyboard:** The header link is a native `<a>` element — keyboard navigable by default (Tab + Enter). No additional ARIA needed in Phase 1.

## CSS File Structure

Per D-03 (external CSS files) and RESEARCH.md Pattern 4:

| File | Component | Scope |
|------|-----------|-------|
| `src/styles/reset.css` | Imported in `devliot-app` | Minimal box-sizing reset + margin reset. Not Shadow DOM — applied globally via `<link>` in `index.html` or via app shell light DOM. |
| `src/styles/app.css` | `devliot-app` shadow root | Three-row flex layout only |
| `src/styles/header.css` | `devliot-header` shadow root | Header padding, link reset |
| `src/styles/footer.css` | `devliot-footer` shadow root | Footer padding, text alignment |
| `src/styles/home.css` | `devliot-home-page` shadow root | Hero section padding |
| `src/styles/article.css` | `devliot-article-page` shadow root | Article wrapper padding |

**CSS custom properties (Phase 1 skeleton):**
```css
:root {
  --color-surface: #ffffff;
  --color-surface-alt: #f5f5f5;
  --color-accent: #0000ee;
  --space-xs: 4px;
  --space-sm: 8px;

  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;
  --space-3xl: 64px;
```
